# ByteSRC 漏洞判定指南

> 用于辅助判断：该漏洞 ByteSRC 收不收、定什么级、有什么注意事项。

---

## 一、不收取 / 忽略的场景

以下情况 ByteSRC **不接收**，AI 发现此类问题时标注 [INFO 不收取] 即可：

- 与模型提示和响应内容相关的问题（jail break、prompt leak、模型幻觉等），除非能证明直接影响机密性/完整性/可用性
- 非信息安全漏洞导致的内容安全风险
- 自身安全产品（如火绒、360、安全 SDK 等）的漏洞
- 非字节（中国区）研发/运维的第三方供应商漏洞
- 火山引擎、BytePlus 外部客户的漏洞
- 个人用户从 Coze 发布至豆包的智能体及插件引发的安全漏洞
- 第三方提供维护的小程序、API 站点、服务器、营销活动、外包系统、抖音电商 ISV
- Hiagent 日常不收取（仅限专测定向邀请）
- 基于网络层面的 DoS、无法影响他人的本地拒绝服务
- 隐私合规相关问题
- 与字节跳动（中国区）业务无关的漏洞
- 用户名遍历、SPF 邮件伪造
- 无任何意义或影响的越权，包括无法影响他人的越权
- 符合业务预期的产品设计
- 内部已知漏洞
- 和安全无关的 Bug 类问题（页面缓慢、样式杂乱等）
- 无法利用或无危害的报告（恶作剧 CSRF、Self-XSS、非敏感信息泄露、dll 劫持、目录遍历等）
- PDF 渲染造成的 XSS
- URL 跳转接口明确用于跳转用途无其他安全危害
- 需注册固定域名来跳转的 URL 跳转
- 不能获取 Cookie 或以用户身份操作的 XSS 且评级不超过中危
- 横向短信轰炸（对不同手机号发送）
- 非隐私数据泄露相关的 AI 服务风险
- 无法稳定复现的漏洞

## 二、各漏洞类型评级标准

### 2.1 远程命令执行 / 代码执行 (RCE)

| 条件 | 评级 |
|------|------|
| 核心业务系统 RCE，可获取大范围权限 | 严重 |
| 单台机器 RCE + 少量信息泄漏 | 高危 |
| 获取不通字节内网的测试系统服务器权限 | 中危1级 |
| 无法获取服务器权限 | 忽略 |
| 沙箱环境代码执行，能逃逸到宿主机/物理机 | 正常评分 |
| 沙箱环境代码执行，不能逃逸 | 视情况 |

### 2.2 SQL 注入

| 条件 | 评级 |
|------|------|
| 核心业务数据库 SQL 注入 | 严重 |
| 非核心业务数据库 SQL 注入 / 少量敏感数据 | 高危 |
| 难以利用的 SQL 注入点（如测试系统） | 低危 |

### 2.3 SSRF

| 条件 | 评级 |
|------|------|
| 可利用的 SSRF（有回显/可交互） | 高危 |
| 通字节内网无回显 SSRF（可差异化响应判断端口） | 中危1级 |
| 只能 SSRF 本机或少量机器，有部分信息泄露 | 不超过中危1级 |
| Blind 类 SSRF（无任何可观察响应差异） | 低危3级 |
| 简单 DNS 日志型 SSRF | 中危 |
| 电商业务无回显 SSRF：PSM 归属电商范畴则收取 | 按实际判定 |
| SSRF 统一按高系数计分 | — |

### 2.4 XSS

| 条件 | 评级 |
|------|------|
| 存储型 XSS | 高危 |
| 仅能执行代码，不能获取 Cookie 或以用户身份操作 | 不超过中危 |
| PDF 渲染造成的 XSS | 不收取 |
| 反射型 XSS | 中危或以下 |

### 2.5 CSRF

| 条件 | 评级 |
|------|------|
| 敏感操作 CSRF + 蠕虫（能绑定手机号和转账） | 高危 |
| 普通 CSRF | 中危或以下 |
| 需明确指出利用链路、传播途径、最终危害 | — |

### 2.6 IDOR / 越权

| 条件 | 评级 |
|------|------|
| 涉及核心业务/重要敏感信息的越权 | 高危 |
| 不涉及核心功能或非重要敏感信息的增删改查越权 | 中危 |
| 电商业务子主账号越权 | 暂不收取 |
| 无任何意义或影响的越权 | 不收取 |
| 同一功能的越权增删改查合并为一个报告提交 | — |

### 2.7 支付逻辑漏洞

| 条件 | 评级 |
|------|------|
| 支付业务逻辑漏洞利用、业务流程绕过 | 高危 |
| 存在定期核帐策略（月度核帐） | 最高不超过中危 |
| 业务预期之内的资损问题、风险可控的问题 | 降级或忽略 |

### 2.8 信息泄露

| 条件 | 评级 |
|------|------|
| 大量核心用户敏感数据泄漏 | 重大 |
| 敏感信息泄漏接口（可获取大量用户敏感信息） | 高危 |
| 轻微信息泄露（路径泄露、.git 泄露、Django Debug、phpinfo、CORS 等） | 中危或低危 |
| 非敏感信息泄露（内网 IP、域名） | 忽略 |

### 2.9 逻辑漏洞 / 业务逻辑缺陷

| 条件 | 评级 |
|------|------|
| 绕过限制使用高级服务/获取优惠（基础版越权获取高级版服务） | 中危 |
| 频控缺陷（短信轰炸需绕过风控，5分钟50条以上同一号码） | 高危 |
| 横向短信轰炸 | 不收取 |
| 反复登出（无限登出） | 中危 |
| 单次登出、Self 型 | 忽略 |
| 发帖、回帖、点赞相关 | 中危、低危或忽略 |
| 一般资料修改（昵称、头像） | 低危或忽略 |
| 用户名遍历 | 不收取 |
| 通过消耗资源造成短时间拒绝服务 | 不超过中危 |

### 2.10 反爬漏洞（生服）

| 条件 | 评级 |
|------|------|
| 日常 | 仅收录严重和高危 |

### 2.11 弱口令

| 条件 | 评级 |
|------|------|
| 以弱口令账户拥有的权限及实际危害评级 | 按实际危害 |
| 需要爆破难度较大的字段（6位及以上验证码、密码） | 降级或忽略 |

### 2.12 移动端漏洞

| 条件 | 评级 |
|------|------|
| 需提供 PoC/Exploit 与验证视频及详尽分析，否则影响评分 | — |
| 客户端远程临时性拒绝服务 | 中危或以下 |
| Auto.js/E4A/hamibot/按键精灵等脚本工具 | 不予收录 |
| 所有根因在 Chromium 的漏洞需字节自研内核启用才有效 | — |

### 2.13 大模型相关产品（豆包、扣子、即梦等）

| 条件 | 评级 |
|------|------|
| 非隐私数据泄露相关的 AI 服务风险（jail break, prompt leak 等） | 不收取 |
| 如可造成额外的可直接验证的安全影响 | 正常收取 |
| 模型提示内容相关问题（越狱、模型幻觉、模型响应恶意内容） | 不收取 |
| 个人用户从 Coze 发布至豆包的智能体及插件引发的漏洞 | 不收取 |

### 2.14 第三方组件 / Nday / 0day

| 条件 | 评级 |
|------|------|
| Nday 已公开 3 个月内且字节已知晓 | 忽略/驳回 |
| Nday 字节不知晓或公开超半年仍未修复 | 根据实际攻击效果评估 |
| 0day 且有可利用证明 | 根据实际攻击效果评估 |
| 需提供 n day 具体链接及 CVE 编号 | — |

### 2.15 图床风险

| 条件 | 评级 |
|------|------|
| 图片永不失效或可通过 ID 接口批量替换达到不失效效果 | 收录 |
| （禁止使用违规图片测试） | — |

## 三、降级场景（以下情况评级降低）

- 利用条件苛刻（特殊账号权限才能利用）
- 需要爆破难度较大的字段（6位及以上验证码、密码、认证信息）
- 部分环节与其他漏洞重复利用
- 支付漏洞存在定期核帐策略，最高不超过中危
- 弱口令按实际拥有权限评级
- 业务预期内的资损问题、风险可控的问题
- 移动端漏洞缺乏 PoC/Exploit 与验证视频
- 沙箱环境代码执行不能逃逸

## 四、情报类

### 4.1 安全情报

| 等级 | 条件 |
|------|------|
| 严重 | 重大影响/大量资金损失，如百万级用户数据泄漏 |
| 高危 | 较大影响/较大资金损失，如有组织薅羊毛 |
| 中危 | 一定影响/一定资金损失，如业务规则缺陷导致批量刷券 |
| 低危 | 轻微影响，如破解版 APK |

### 4.2 情报收取范围

- 用户敏感数据大规模泄露 + 攻击代码/利用工具
- 服务器被入侵 + 入侵行为特征
- 刷量行为关键线索（关注、分享、点赞、评论等）
- 支付类作弊关键线索（iOS 代退款等）
- 游戏类作弊关键线索（外挂、练号打金等）
- 直播类作弊关键线索（刷人气、抢红包等）
- 批量恶意账号注册、有组织薅羊毛
- 内部信息/用户数据泄露（网盘、GitHub 等）
- 蠕虫传播、流量劫持关键线索

## 五、评分与奖励

- 报告质量加分（1-50分）：结构规范、测试账号、利用思路新颖
- 处罚：多次自评虚高、多次无效报告、AI 生成未验证
- 累计 3 个无效报告提醒，5 个封禁
- 多漏洞合并：按利用链最终危害评分
- 同类漏洞 60 天内前三个有效
- 最早提交者优先奖励
